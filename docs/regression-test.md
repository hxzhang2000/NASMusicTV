## 1. 测试概述

### 1.1 测试目标

本文档定义了 NAS Music TV 应用的完整回归测试用例，确保每次版本迭代后所有功能正常工作，特别覆盖以下高风险区域：

- **后端适配器**（Jellyfin / Navidrome）API 调用正确性
- **播放器生命周期**与资源释放
- **分页加载**与增量映射构建
- **UI 焦点系统**（FocusableSurface 组件）
- **退出清理**与进程终止
- **密码加密**存储
- **网络重连**机制

### 1.2 测试分类

| 分类 | 说明 | 执行方式 |
|------|------|----------|
| 单元测试 | 工具类、解析器、状态机逻辑 | `./gradlew test` 自动执行 |
| 集成测试 | 后端 API 调用、播放器集成 | 手动 + 半自动 |
| UI 测试 | 焦点导航、屏幕交互 | 手动（TV 遥控器） |
| 专项验证 | 近期修复项的定向验证 | 手动 |

### 1.3 优先级定义

| 优先级 | 说明 |
|--------|------|
| **P0** | 阻断核心功能，必须通过 |
| **P1** | 影响重要功能，应通过 |
| **P2** | 边界场景或体验问题，建议通过 |

## 2. 测试环境准备

### 2.1 硬件环境

- Android TV 设备（API 22+，建议 API 33+）
- D-Pad 遥控器（支持方向键、OK 键、BACK 键、媒体键）
- 网络连接（Wi-Fi 或有线）

### 2.2 软件环境

- NAS Music TV v2.2.0+（Debug 构建）
- Jellyfin 服务器（含音乐库、歌词、封面）
- Navidrome 服务器（含音乐库）

### 2.3 测试数据准备

| 数据项 | 要求 |
|--------|------|
| 音乐库 | 至少 500 首歌曲，覆盖多专辑、多歌唱家、多流派、多年份 |
| 歌词 | 至少 20 首歌曲有内嵌 LRC 歌词，5 首有 Jellyfin 服务端歌词 |
| 封面 | 大部分专辑有封面图，部分歌曲无封面（测试 fallback） |
| 多歌唱家 | 至少 10 首歌曲含 "A feat. B" 或 "A & B" 格式 |
| 年份分布 | 歌曲生产年份覆盖 1990s-2020s |
| 收藏 | 测试前在服务端预设 5-10 首已收藏歌曲 |

### 2.4 服务器配置

```
Jellyfin 服务器：
- URL: http://<jellyfin-host>:8096
- 用户名/密码：测试账号
- 确保有 API Token 或可用户名密码登录

Navidrome 服务器：
- URL: http://<navidrome-host>:4533
- 用户名/密码：测试账号
```
